Documentation

Understanding the bridge between your Android device and Windows environment.

01 How it Works

The bridge operates as a high-speed relay. The Android client captures input data (including pressure, tilt, and coordinates) and packages it into optimized UDP packets sent to the desktop server.

  • Step 1: Server initializes a listener on a specific UDP port.
  • Step 2: Client sends a handshake request to the server.
  • Step 3: Handshake is established, and low-latency input streaming begins.

02 Discovery

Discovery is what makes the connection process actually feel smooth. Instead of forcing someone to type the IP and Port manually the app itself figures out what's nearby. It's a pretty simple process:

  • Step 1: The client sends out a discovery request. It's basically a broadcast asking, "Hey, is anyone there? (server)".
  • Step 2: Any server that sees that packet sends a response with its info.
  • Step 3: The client collects those responses into a nice list so the user can just pick which one they want to join.

03 Logging & Debugging

The server generates logs in the console. These logs are vital for troubleshooting errors.
Level of logging can be overriden by --log-level (level)

Server Usage

The server can be launched with arguments for automation or headless setups:

--log-level [number/option]

Override the logger severity level. (silent-verbose)

--help

Lists all arguments you can use on your server version.
