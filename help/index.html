Bridge Diagnostic Center

Pro Tip:

Most of discovery issues are fixed by restarting both the Android App and the Server Executable.

Wrong Wi-Fi

Ensure that the server and client are on the same Wi-Fi, also check if the computer has Wi-Fi turned on.

Network Profile

Ensure your Windows Network is set to Private. Public profiles block the UDP broadcasts needed for discovery.

VPN / Proxy

Active VPNs create virtual adapters that confuse discovery. Disable them while using the bridge.

Permissions

Check Android "Nearby Devices" or "Local Network" permissions if prompted. Check Windows Firewall logs.

Server & Hardware Conflicts

Issue Category Symptom Recommended Action
Input Drivers Pressure/Tilt not working Switch from Wintab to Windows Ink in settings.
Multi-Monitor Drawing on wrong screen App targets the Primary Monitor. Verify primary display in Windows Settings or report if dynamic swap fails.
OS Version Service won't start Windows 10/11 required. Check server console for specific Kernel Errors.
Android System Sudden disconnects Disable "Battery Optimization" for the app. Android may be killing the background process.

Check Server Console

If the server encounters a critical error, it will log the details in the terminal window. Copy the terminal lines before reporting.
